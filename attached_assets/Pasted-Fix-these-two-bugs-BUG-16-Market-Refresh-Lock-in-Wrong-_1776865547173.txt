BUG #2 — Edge Badge Color (persistent)

Edge of +4.0 shows no color badge on the scanner.

In artifacts/alpha-lens/src/components/ui-helpers.tsx,
find the ScoreDisplay component or edge badge logic.

Most likely cause: Tailwind class names being
constructed dynamically (e.g. bg-${color}) which
get purged at build time and never apply.

Fix using hardcoded Tailwind class strings only:

const edgeNum = Number(edge ?? 0);
const badgeColor = edgeNum > 0
  ? "bg-green-500/10 text-green-400 border-green-500/20"
  : edgeNum < 0
  ? "bg-red-500/10 text-red-400 border-red-500/20"
  : "bg-muted text-muted-foreground border-border";

Never construct Tailwind class names dynamically
from variables — always use complete hardcoded
class strings.

Also add explicit type coercion in case edge
arrives as a string:
  const edgeNum = Number(edge ?? 0);

Run pnpm run typecheck. Zero errors required.

Verify on frontend: asset with edge +4.0 shows
green badge. Edge -3.2 shows red. Edge 0 shows gray.
