Add BLS (Bureau of Labor Statistics) macro data to the
existing macro-data.ts service in
artifacts/api-server/src/services/macro-data.ts

BLS API documentation is confirmed. Use these exact
endpoints:

BASE URL: https://api.bls.gov/publicAPI/v2/timeseries/data/
METHOD: POST
HEADER: Content-Type: application/json
API KEY ENV VAR: process.env.BLS_API_KEY

STEP 1 — Add fetchCPI() function

POST to BLS API with this payload:
{
  "seriesid": ["CUUR0000SA0"],
  "startyear": "currentYear - 1",
  "endyear": "currentYear",
  "registrationkey": process.env.BLS_API_KEY || ""
}

Series CUUR0000SA0 = Consumer Price Index,
All Urban Consumers, All Items.

Parse response:
  Results.series[0].data — sorted by year+period
  descending. Take the two most recent entries.
  current = parseFloat(data[0].value)
  previous = parseFloat(data[1].value)
  change = parseFloat(
    (((current - previous) / previous) * 100).toFixed(2)
  )
  date = data[0].year + "-" + data[0].period

Return:
{
  current,
  previous,
  change,
  date,
  source: "BLS"
}

On any error: return null. Never throw.
Log: "BLS CPI: {current} (change: {change}%)"
On failure: "BLS CPI: fetch failed"

STEP 3 — Add fetchBLSMacro() function

Fetch both series in a single POST request
for efficiency:
{
  "seriesid": ["CUUR0000SA0", "LNS14000000"],
  "startyear": "currentYear - 1",
  "endyear": "currentYear",
  "registrationkey": process.env.BLS_API_KEY || ""
}

Parse both series from the response.
Use Promise pattern — single API call,
split results by seriesID.

Return:
{
  cpi: { current, previous, change, date, source },
  unemployment: { current, previous, change, date, source }
}

Add 24-hour cache — BLS data is monthly,
no need to fetch more than once per day.
Short TTL (5 min) on failures.
