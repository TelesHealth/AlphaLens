FIX 1 — Audit item 7.3
Radar scan lock race condition between scheduler
and manual route.

In artifacts/api-server/src/routes/radar.ts,
find the module-level scanInProgress lock
(around line 23).

Remove it entirely from the route file.

The route handler for POST /api/radar/scan
should simply call runRadarScan() from the
market-radar service and return the result.

The existing isRadarScanning lock in
scheduler.ts (line 8) is the single source
of truth. runRadarScan() in market-radar.ts
must also check and set that same lock before
running — not a separate lock.

Verify the shared lock is in market-radar.ts
or accessible from both the scheduler and the
route. After fix, a manual POST /api/radar/scan
while the scheduler is running should return
{ status: "scan_already_running" } not start
a second scan.
