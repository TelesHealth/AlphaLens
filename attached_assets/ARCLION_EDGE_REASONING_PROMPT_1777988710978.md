## Context

The platform currently explains WHY the AI likes a trade
through the `why` array and `headline` fields. But it does
NOT explicitly explain the edge number itself, the
confidence level, or alert users when edge has changed
since the original call.

This prompt adds three improvements:
1. Explicit edge explanation sentence per recommendation
2. Confidence rationale sentence per recommendation
3. Edge change alert when edge narrows or widens

## CHANGE 1 — Add edgeExplanation field

### Schema update

In lib/db/src/schema/, add to recommendations table:

  edgeExplanation: text nullable
    — one sentence explaining why the edge exists
    — generated by Claude at scan time
    — directly references the edge number and data sources

  confidenceRationale: text nullable
    — one sentence explaining why confidence is high/low
    — references the data sources that support or
      undermine the call

  edgePrevious: numeric nullable
    — the edge value from the previous scan for this asset
    — used to calculate edge change alerts
    — null on first scan for this asset

  edgeChangedAt: timestamp nullable
    — when the edge last changed significantly (>5 points)

Run: pnpm --filter @workspace/db run push

## CHANGE 2 — Track edge changes between scans

### In refreshRecommendationEdges() function

This function was added in the Edge Calculation
Improvements prompt. It runs every 5 minutes after
each market data refresh.

Update it to also track edge changes:

  For each open recommendation being updated:

  1. Read the current edge value before updating:
     const previousEdge = rec.edge;

  2. Calculate the new edge as before.

  3. Calculate the edge change:
     const edgeChange = Math.abs(newEdge - previousEdge);

  4. If edgeChange >= 5 (significant change):
     Set edgePrevious = previousEdge
     Set edgeChangedAt = new Date()
     Log: "Edge change: rec #${id} ${rec.assetTitle}
       edge moved from ${previousEdge} to ${newEdge}
       (${edgeChange > 0 ? '+' : ''}${newEdge - previousEdge})"

  5. If edgeChange < 5: do not update edgePrevious
     or edgeChangedAt (noise filtering)

  6. Always update edge, convictionScore, marketPrice,
     and edgeCalculatedAt regardless of change size.

## CHANGE 3 — Add edgeChangeAlert to API responses

In artifacts/api-server/src/routes/recommendations.ts,
in the GET /api/recommendations/briefing endpoint,
add an edgeChangeAlert field to each recommendation
in the response:

  // Calculate edge change alert
  const edgeChangeAlert = (() => {
    if (!rec.edgePrevious || !rec.edgeChangedAt) return null;

    const change = rec.edge - rec.edgePrevious;
    const minutesAgo = Math.floor(
      (Date.now() - rec.edgeChangedAt.getTime()) / 60000
    );

    // Only show alert if change was recent (last 2 hours)
    if (minutesAgo > 120) return null;

    const direction = change > 0 ? "widening" : "narrowing";
    const sign = change > 0 ? "+" : "";

    return {
      hasAlert: true,
      direction,
      change: Math.round(change * 10) / 10,
      previousEdge: rec.edgePrevious,
      currentEdge: rec.edge,
      minutesAgo,
      message: `Edge ${direction}: was ${rec.edgePrevious} pts,
        now ${rec.edge} pts (${sign}${Math.round(change * 10) / 10}
        in last ${minutesAgo} min)`
    };
  })();

Add edgeChangeAlert to the recommendation response object.
If null, omit it from the response (don't send null field).
